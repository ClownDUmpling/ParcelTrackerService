# Parcel Tracker Service (Go + SQLite)
## Описание проекта
Parcel Tracker Service — это минималистичный бэкенд-сервис для регистрации, управления и отслеживания посылок. Сервис написан на языке Go и использует базу данных SQLite для хранения информации о посылках, их статусах и адресах клиентов.

Проект демонстрирует реализацию базовых CRUD-операций и бизнес-логики (например, ограничение на изменение адреса или удаление посылки в зависимости от статуса) с использованием чистой архитектуры.

## Стек технологий
Язык: Go (Golang 1.23+)

База данных: SQLite (через драйвер modernc.org/sqlite)

Контейнеризация: Docker

Тестирование: testing package, github.com/stretchr/testify/require

CI/CD: GitHub Actions (сборка, тестирование, деплой Docker-образа)

## Функциональность
### Сервис предоставляет следующие ключевые возможности, реализованные в пакете main:

- Регистрация посылки (Register): Добавление новой посылки с указанием клиента и адреса, установка статуса "registered".

- Изменение адреса (ChangeAddress): Смена адреса доставки, разрешенная только для посылок в статусе "registered".

- Изменение статуса (NextStatus): Перевод посылки в следующий статус ("registered" → "sent" → "delivered").

- Удаление посылки (Delete): Удаление посылки, разрешенное только в статусе "registered".

- Просмотр посылок клиента (PrintClientParcels): Получение и вывод всех посылок конкретного клиента.

## Запуск проекта
Вы можете запустить проект локально с помощью Go или использовать Docker для контейнеризации.

### 1. Локальный запуск (Go)
1. Убедитесь, что у вас установлен Go (версия 1.23 или выше).
2. Клонируйте репозиторий:

`git clone [ссылка на ваш репозиторий]`  
`cd [имя вашего репозитория]`  

3. Запустите основное приложение (оно содержит демонстрационный сценарий в main.go):

`go run main.go`  

### 2. Запуск через Docker
1. Убедитесь, что у вас установлен Docker.

2. Соберите Docker-образ, используя предоставленный Dockerfile:

`docker build -t parcel-tracker-service .`  
3. Запустите контейнер:

`docker run --rm parcel-tracker-service`  
## Тестирование
Для проверки корректности работы хранилища и бизнес-логики предусмотрены модульные тесты в файле parcel_test.go.

Для запуска тестов используйте стандартную команду Go:

`go test -v`

## CI/CD (GitHub Actions)
В репозитории настроен пайплайн (описанный в go.yaml), который автоматически:

1. Тестирует код при каждом push в репозиторий.

2. Проводит статический анализ кода (go vet).

3. Собирает и деплоит Docker-образ на Docker Hub при создании нового тега (например, v1.0.0).
